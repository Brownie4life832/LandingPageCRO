# Audience Rewrite

Take an existing landing page and rewrite its copy for a specific audience persona. Same product, different story.

## Input

The user provides:
- **The page**: URL, file, screenshot, OR a reference to a previously analyzed page
- **The target persona**: A specific audience segment (e.g., "seniors aged 65+", "busy working moms", "wheelchair users", "budget-conscious college students")

If the user doesn't specify a persona, identify 2-3 distinct personas from the page and ask which one to write for. If the page clearly targets multiple audiences (like the stepper targeting professionals, elderly, and wheelchair users), offer to write for each.

## Rewrite Workflow

### Step 1: Understand the Current Page

If not already analyzed in this session:
- Load the page and capture key content
- Identify: current headline, subheadline, value props, CTA copy, benefit statements, social proof, and overall tone

If the page was already analyzed, reference the existing analysis — don't re-analyze.

### Step 2: Build the Persona Profile

Before writing, define the persona in detail:

**[Persona Name]**
- **Who they are**: Age range, life situation, daily reality
- **Primary pain**: The #1 problem this product solves for them (may differ from other personas)
- **Emotional trigger**: What feeling drives their purchase — fear, hope, frustration, aspiration?
- **Objections**: What would stop THIS person from buying? (different personas have different objections)
- **Language**: How do they talk about this problem? What words do they use?
- **Decision driver**: Price? Trust? Ease of use? Speed? Social validation?
- **Where they are in awareness**: Do they know the product category exists? Do they know they have this problem?

### Step 3: Rewrite the Full Page Copy

Rewrite every key copy element for this specific persona:

1. **Headline** — Speaks directly to THIS persona's primary pain or desired outcome
2. **Subheadline** — Expands with details relevant to THEIR life
3. **Hero section framing** — What image/context would resonate with them?
4. **Value prop blocks** (top 4-6) — Same product features, reframed for THIS persona's priorities. Reorder by what matters most to them.
5. **Social proof angle** — What kind of testimonial would this persona trust? Write an example.
6. **Objection handling** — Address THIS persona's specific hesitations
7. **CTA copy** — Language that feels natural to this audience
8. **Narrative arc** — Full 5-part story (Problem → Shift → Solution → Proof → Close) written for THIS persona

Show a side-by-side of key copy elements:

| Element | Current Copy | Rewritten for [Persona] |
|---------|-------------|------------------------|
| Headline | "..." | "..." |
| Subheadline | "..." | "..." |
| Primary CTA | "..." | "..." |

### Step 5: Implementation Notes

- **Can this run as a variant of the same page?** (A/B test by audience segment)
- **Or does it need a separate landing page?** (different entry points for different audiences)
- **Ad copy suggestions**: 1-2 ad headlines that would drive THIS persona to this rewritten page
- **Recommended targeting**: Where does this persona hang out? What search terms do they use?

## Guidelines

- The product doesn't change. The story does. Same features, different emphasis, different emotional entry point.
- Don't water down the copy to be "inclusive of everyone." The whole point is to speak to ONE person. Specificity converts.
- Use the persona's actual language. If you're writing for a 72-year-old, don't use startup jargon. If you're writing for a Gen Z audience, don't sound corporate.
- The rewrite should feel like a completely different page, even though it's selling the same thing. If you swap the persona name and the copy still reads the same, you haven't gone deep enough.
- When writing example testimonials, make them feel real — specific details, specific outcomes, imperfect language.
